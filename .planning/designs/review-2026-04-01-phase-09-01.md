## Critical Finding: AddPropertyDialog Depends on nodeMenu

The plan's original Task 1 approach (microtask delay to close menu before dialog opens) would have created a new bug. Line 644 of OntologyCanvas.tsx has `{addPropFor && nodeMenu && (` — the dialog requires `nodeMenu` to be non-null. Closing the menu first means the dialog never renders.

**Resolution:** Decouple dialog from nodeMenu state. Expand `addPropFor` to include `nodeType`, remove `nodeMenu` dependency.
