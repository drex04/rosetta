# Plan 16-01: Onboarding, Edge & MAP Tab Bug Fixes

## Context

Nine bugs identified after Phase 13 completion. All are targeted fixes with clear root causes — no new architecture needed.

**Root cause summary:**
- `onStartFresh` never resets stores → seeded ontology persists
- Tour starts while modal may still be in DOM → beacon fallback
- `EdgeContextMenu` passes React Flow edge ID (`mapping_abc123`) to `updateMapping`/`removeMapping` but store expects bare mapping ID (`abc123`)
- `updateEdge` patches `label` at the React Flow top-level edge but `ObjectPropertyEdge` reads `data?.label`
- `MappingEdge.KIND_LABEL` uses abbreviations; user wants full names matching the MAP tab select
- MAP tab `FormulaTier` includes `'rml'` tier that user wants removed
- Delete "X" button uses `text-muted-foreground` by default — not visually destructive
- `FormBuilder.localFn` state defaults to `'CONCAT'` on remount — not derived from expression when expression is empty
- Example project only seeds Norway mappings

## Tasks

### Task 1 — Fix "Start Fresh": reset all stores
**File:** `src/App.tsx`

`onStartFresh` currently only writes to localStorage and closes the modal. Add store resets before closing:

```ts
onStartFresh={() => {
  useOntologyStore.getState().reset();
  useSourcesStore.getState().reset();
  useMappingStore.getState().reset();
  useValidationStore.getState().reset();
  localStorage.setItem('rosetta-tour-seen', '1');
  setShowModal(false);
}}
```

Import the missing stores if not already imported in App.tsx.

**Done:** Clicking "Start Fresh" leaves an empty canvas with no classes, no sources, no mappings.

### Task 2 — Fix tour auto-start: close modal before starting tour
**File:** `src/App.tsx`

Move `setShowModal(false)` to fire immediately when the button is clicked (before `await loadExampleProject()`), so the modal overlay is fully gone before joyride initialises. Currently both calls happen in the same batched render after the async `await`, which risks joyride finding the modal overlay still present and falling back to beacon mode.

```ts
onLoadExample={async () => {
  if (loadingExample) return;
  setLoadingExample(true);
  setShowModal(false);              // ← close modal first, synchronously
  try {
    await loadExampleProject();
    setTourRunning(true);
  } catch (err) {
    console.error('[loadExampleProject] failed:', err);
  } finally {
    setLoadingExample(false);
  }
}}
```

**Done:** After clicking "Load Example & Start Tour", the tour tooltip appears immediately on the first target element with no beacon.

### Task 3 — Fix EdgeContextMenu: use mappingId not edgeId for store calls
**File:** `src/components/canvas/EdgeContextMenu.tsx`

React Flow mapping edge IDs are `mapping_${mapping.id}` (see `useCanvasData.ts:56`). The mapping store stores mappings under their bare `id`. Every `updateMapping(menu.edgeId, ...)` and `removeMapping(menu.edgeId)` call is passing `mapping_abc123` instead of `abc123`, so nothing happens.

Fix: use `(menu.edgeData as { mappingId: string }).mappingId` for all mapping store calls:

```ts
// kind change
updateMapping((menu.edgeData as { mappingId: string }).mappingId, { kind });

// delete
removeMapping((menu.edgeData as { mappingId: string }).mappingId);
```

The `mappingId` field is already stored in edge data (`useCanvasData.ts:63`).

**Done:** Right-clicking a mapping edge and selecting a different kind immediately updates the edge label and the kind picker in the MAP tab.

### Task 4 — Fix ObjectProperty edge label: patch data.label not top-level label
**File:** `src/components/canvas/EdgeContextMenu.tsx`

`updateEdge(id, { type: 'objectPropertyEdge', label: pendingLabel })` sets the React Flow top-level `edge.label` string. `ObjectPropertyEdge` reads `data?.label`. These are different fields — the label never appears.

Fix: put the label inside `data`:

```ts
// Change to Object Property (subclassEdge → objectPropertyEdge)
updateEdge(menu.edgeId, {
  type: 'objectPropertyEdge',
  data: { ...menu.edgeData, label: pendingLabel.trim() },
});

// Change to Subclass (objectPropertyEdge → subclassEdge)
updateEdge(menu.edgeId, {
  type: 'subclassEdge',
  data: { ...menu.edgeData, label: '' },
});
```

**Done:** Creating an ObjectProperty edge shows the user-typed label. Switching back to Subclass shows "subClassOf" (hardcoded in SubclassEdge). No label disappears.

### Task 5 — Full mapping edge kind labels (no abbreviations)
**File:** `src/components/edges/MappingEdge.tsx`

Remove the `KIND_LABEL` abbreviation map. Use the `kind` string directly as the label. Remove stale `join` and `sparql` entries that no longer exist.

Before:
```ts
const KIND_LABEL: Record<string, string> = {
  direct: 'direct',
  template: 'tmpl',
  constant: 'const',
  typecast: 'cast',
  language: 'lang',
  join: 'join',
  sparql: 'sparql',
};
const kindLabel = kind ? (KIND_LABEL[kind] ?? kind) : null;
```

After:
```ts
const kindLabel = kind ?? null;
```

**Done:** Mapping edge labels read "template", "constant", "typecast", "language" — matching the MAP tab kind picker exactly.

### Task 8 — Fix FormBuilder funcType reset on tab switch
**File:** `src/components/panels/MappingPanel.tsx`

`FormBuilder` uses `const [localFn, setLocalFn] = useState<string>(derived?.fn ?? 'CONCAT')`. When `formulaExpression` is empty (user has picked a function but not selected all fields yet), `derived` is null and `localFn` defaults to `'CONCAT'` on every remount.

Fix: lift `localFn` out of `FormBuilder` into `MappingDetail` so it persists across tier switches:

1. Add `const [formulaFn, setFormulaFn] = useState<string>(() => derived?.fn ?? 'CONCAT')` to `MappingDetail` (compute `derived` from `mapping.formulaExpression` using the same logic already in FormBuilder)
2. Pass `fn={formulaFn}` and `onFnChange={setFormulaFn}` as props to `FormBuilder`
3. Remove the `localFn`/`setLocalFn` state from `FormBuilder`; use the props instead

**Done:** Selecting UPPER in the Form tab, switching to Formula, and switching back still shows UPPER — not CONCAT.

**Done:** Loading the example shows mapping edges for both Norway and Germany sources. Switching source tabs shows pre-seeded mappings for each.
