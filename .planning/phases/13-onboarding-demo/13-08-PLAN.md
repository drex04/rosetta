# Plan 13-08: Node Search — Always Visible, Property Search, Richer Highlights

## Context

Node search was built in 13-06 as a Ctrl+F-toggled floating panel (top-center). User feedback requests:
- Always show the search field (top-right corner)
- Search properties too, not just node labels/URIs
- Bolder highlight ring on matching nodes
- Highlight the specific matching property row within the node

Current implementation: `searchOpen` boolean gates a `<Panel position="top-center">`. `matchingIds` is a `Set<string>` keyed by node id, matched against `data.label` and `data.uri`. Nodes receive `isSearchHighlighted: boolean` in their data. Ring style: `ring-2 ring-blue-400 ring-offset-1`.

## Tasks

### Task 1 — Always-visible search panel (top-right)

**File:** `src/components/canvas/OntologyCanvas.tsx`

Remove `searchOpen` and `setSearchOpen` state entirely. The search Panel is always rendered (no conditional wrapper).

Change Panel position from `"top-center"` to `"top-right"`. The existing `top-left` Panel (minimap controls) is not affected.

Add a `searchInputRef = useRef<HTMLInputElement>(null)` and forward it to the input element. Pass this ref up to `useKeyboardShortcuts` (either as a prop or via a ref callback forwarded through the existing `onOpenSearch` mechanism).

Update Escape handling: `onKeyDown={(e) => { if (e.key === 'Escape') setSearchQuery(''); }}` — clears the query but does not hide the panel (there is no panel to hide).

Remove the `Ctrl+F → setSearchOpen(true)` logic from `useKeyboardShortcuts.ts` — replace with `searchInputRef.current?.focus()`.

The panel HTML:
```tsx
<Panel position="top-right" className="mt-2 mr-2">
  <div className="flex items-center gap-1.5 bg-background border border-border rounded-md shadow-md px-2 py-1">
    <MagnifyingGlassIcon size={14} className="text-muted-foreground shrink-0" />
    <input
      ref={searchInputRef}
      value={searchQuery}
      onChange={(e) => setSearchQuery(e.target.value)}
      onKeyDown={(e) => { if (e.key === 'Escape') setSearchQuery(''); }}
      placeholder="Search nodes…"
      className="bg-transparent text-sm outline-none w-40 placeholder:text-muted-foreground"
    />
    {searchQuery && (
      <span className="text-xs text-muted-foreground shrink-0">
        {matchingIds.size} found
      </span>
    )}
  </div>
</Panel>
```

**Done:** The search input is visible at all times in the top-right corner. Ctrl+F focuses it. Escape clears the query. No toggle exists.

### Task 2 — Property search and matchedPropUrisMap

**File:** `src/components/canvas/OntologyCanvas.tsx`

Extend the `matchingIds` derivation (currently `useMemo`) to also match nodes whose `data.properties` array contains any property where `prop.label` or `prop.uri` includes the query string (case-insensitive).

Alongside `matchingIds: Set<string>`, derive a `matchedPropUrisMap: Map<string, string[]>` — keyed by node id, values are the property URIs that matched. A node appears in `matchingIds` if either its own label/URI matches OR it has at least one matched property.

```ts
const { matchingIds, matchedPropUrisMap } = useMemo(() => {
  const matchingIds = new Set<string>();
  const matchedPropUrisMap = new Map<string, string[]>();

  if (!searchQuery.trim()) return { matchingIds, matchedPropUrisMap };
  const q = searchQuery.toLowerCase();

  for (const node of nodes) {
    const d = node.data as { label?: string; uri?: string; properties?: { uri: string; label?: string }[] };
    const labelHit = d.label?.toLowerCase().includes(q);
    const uriHit = d.uri?.toLowerCase().includes(q);
    const propHits = (d.properties ?? [])
      .filter((p) => p.uri.toLowerCase().includes(q) || (p.label ?? '').toLowerCase().includes(q))
      .map((p) => p.uri);

    if (labelHit || uriHit || propHits.length > 0) {
      matchingIds.add(node.id);
      if (propHits.length > 0) matchedPropUrisMap.set(node.id, propHits);
    }
  }
  return { matchingIds, matchedPropUrisMap };
}, [nodes, searchQuery]);
```

**Done:** `matchedPropUrisMap` contains per-node arrays of matched property URIs. Nodes match on property content, not just their own label/URI.

### Task 3 — Pass matchedPropUris and bolder ring into node data

**File:** `src/components/canvas/OntologyCanvas.tsx`

In the node data population (around line 766), update the `isSearchHighlighted` and add `matchedPropUris`:

```ts
isSearchHighlighted: searchQuery.trim().length > 0 && matchingIds.has(n.id),
matchedPropUris: matchedPropUrisMap.get(n.id) ?? [],
```

**Files:** `src/components/nodes/ClassNode.tsx` and `src/components/nodes/SourceNode.tsx`

Both nodes already read `isSearchHighlighted` from `data`. Change the ring class from:
```tsx
isSearchHighlighted ? 'ring-2 ring-blue-400 ring-offset-1' : '',
```
to:
```tsx
isSearchHighlighted ? 'ring-2 ring-primary ring-offset-2' : '',
```

(Use `ring-primary` for design-system consistency rather than a hardcoded blue.)

In the properties map section of each node, read `matchedPropUris`:
```tsx
const matchedPropUris: string[] = (data.matchedPropUris as string[] | undefined) ?? [];
```

For each property row, add a highlight class when `matchedPropUris.includes(prop.uri)`:
```tsx
<div
  key={prop.uri}
  className={cn(
    'flex items-center gap-1 px-1 py-0.5 text-sm rounded',
    matchedPropUris.includes(prop.uri) ? 'bg-primary/10' : '',
  )}
  ...
>
```

**Done:** Matching nodes show a bolder primary-colored ring. Rows for matched properties get a subtle `bg-primary/10` background. Non-matched property rows are unaffected.

### Task 4 — Update useKeyboardShortcuts for ref-based focus

**File:** `src/hooks/useKeyboardShortcuts.ts`

The hook currently accepts `onOpenSearch?: () => void` and calls it on Ctrl+F. Change the param to accept a `searchInputRef: React.RefObject<HTMLInputElement>` instead:

```ts
interface ShortcutOptions {
  searchInputRef: React.RefObject<HTMLInputElement>;
  // ... existing options
}
```

In the Ctrl+F handler:
```ts
if (e.ctrlKey && e.key === 'f') {
  e.preventDefault();
  searchInputRef.current?.focus();
  searchInputRef.current?.select();
  return;
}
```

Update the call site in `OntologyCanvas.tsx` to pass `searchInputRef` instead of a `setSearchOpen` callback.

**Done:** Ctrl+F focuses the always-visible search input. No toggle behavior remains. TypeScript passes on `npm run build`.
