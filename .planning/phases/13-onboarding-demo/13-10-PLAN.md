# Plan 13-10: Onboarding Improvements

## Context

Phase 13-07 shipped react-joyride with auto-start on first visit and a "Tour" button in the Header. User feedback requests a revised onboarding flow:

- **Modal-first**: Show a welcome modal before the tour, not an immediate auto-start.
- **Load example before tour**: The modal's primary CTA loads sample data so the tour makes sense.
- **Tab switching**: Each tour step should open the relevant right-panel tab.
- **Tour step fixes**: Reorder, remove SPARQL mention, add final step, remove beacon dots.
- **Header**: Help button opens tour; Tour button is removed.

Key existing facts:
- `uiStore` has `tourRunning` (currently initializes to `!localStorage.getItem('rosetta-tour-seen')`) and `setTourRunning`.
- `uiStore` has `activeRightTab: RightTab` and `setActiveRightTab`.
- Tour steps currently: load → add-source → ontology → canvas → map → validate → output (7 steps). Transform/output is last; should be second-to-last (before validate).
- `TourProvider.tsx` calls `setTourRunning(false)` on `STATUS.FINISHED | STATUS.SKIPPED` and sets `rosetta-tour-seen`.
- Header has both a "Tour" button (`setTourRunning(true)`) and a separate "Help" button.

## Tasks

### Task 1 — loadExampleProject() utility

**File:** `src/lib/exampleProject.ts` (new file)

Read `e2e/helpers.ts` first to understand how the E2E `loadExampleProject` sets up application state (which file upload or import action it triggers). Then implement the app-level equivalent.

The utility should:
1. Reset all stores to empty state (`ontologyStore.reset()`, `sourcesStore.reset()`, `mappingStore.reset()`, `fusionStore.reset()` — check each store for a `reset` action).
2. Load the sample ontology from `src/data/sample-ontology.ttl` (dynamic import or `fetch` from the public path). Call the appropriate ontology hydrate/import action.
3. Load the two sample JSON sources (`sample-source-a-norwegian.json`, `sample-source-b-german.json`) and the XML source (`sample-source-c-uk.xml`). For each, call the store action that adds a source from raw text (the same action the "Add Source" flow uses).
4. If sample mappings exist in `src/data/` (check for a `.json` project bundle), load them via `mappingStore.hydrate(...)`. Otherwise, skip — the tour will show an empty MAP tab which is still informative.

Export as a plain async function:
```ts
export async function loadExampleProject(): Promise<void> { ... }
```

**Implementation note:** If the E2E helper uses a project file import (dragging a `.rosetta.json` bundle), there is likely a `importProject(projectJson)` utility already in the codebase. Find it and reuse it — this avoids duplicating hydrate sequencing. If no such utility exists, sequence the individual store hydrate calls manually.

**Done:** `await loadExampleProject()` resolves and the canvas shows ontology nodes, source nodes, and (if available) mapping edges — the same state as after running the E2E `loadExampleProject` helper.

### Task 4 — Update TourProvider for tab switching

**File:** `src/components/onboarding/TourProvider.tsx`

Add a step-index to tab mapping:
```ts
const TAB_FOR_STEP: Record<number, RightTab | null> = {
  0: 'SOURCE',
  1: 'SOURCE',
  2: 'ONTOLOGY',
  3: null,       // canvas step — no tab switch
  4: 'MAP',
  5: 'OUTPUT',
  6: 'VALIDATE',
  7: null,       // final step — no tab switch
};
```

Import `useUiStore` and pull in `setActiveRightTab`.

In `handleEvent`, add a branch for `type === 'step:before'`:
```ts
function handleEvent(data: EventData) {
  const { status, type, index } = data;

  if (type === 'step:before') {
    const tab = TAB_FOR_STEP[index as number];
    if (tab) setActiveRightTab(tab);
  }

  if (status === STATUS.FINISHED || status === STATUS.SKIPPED) {
    localStorage.setItem(TOUR_KEY, '1');
    setTourRunning(false);
  }
}
```

Note: `STATUS.FINISHED` and `STATUS.SKIPPED` are the only events that set `rosetta-tour-seen`. "Start Fresh" (modal dismiss) does NOT set this flag — confirmed design.

**Done:** Each tour step triggers a right-panel tab switch. The correct tab is active when the step's tooltip appears. Tour completion or skip sets the localStorage flag and prevents the modal from re-appearing.
