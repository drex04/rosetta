# Plan 13-06: Canvas UX — Node Search, Shortcuts, Tooltips

## Context

Four UX improvements: a searchable canvas, global keyboard shortcuts, tooltips on key controls, and consistent loading/error feedback for Transform & Fuse.

## Tasks

### Task 1 — Node search on canvas

**File:** `src/components/canvas/OntologyCanvas.tsx`

Add a `searchQuery` state (`useState<string>('')`) and a `searchOpen` state (`useState<boolean>(false)`).

**Search Panel:** Add a `<Panel position="top-center">` (React Flow Panel component) that renders only when `searchOpen`:

```tsx
{searchOpen && (
  <Panel position="top-center" className="mt-2">
    <div className="flex items-center gap-1.5 bg-background border border-border rounded-md shadow-md px-2 py-1">
      <MagnifyingGlassIcon size={14} className="text-muted-foreground shrink-0" />
      <input
        autoFocus
        value={searchQuery}
        onChange={(e) => setSearchQuery(e.target.value)}
        onKeyDown={(e) => { if (e.key === 'Escape') { setSearchOpen(false); setSearchQuery(''); } }}
        placeholder="Search nodes…"
        className="bg-transparent text-sm outline-none w-48 placeholder:text-muted-foreground"
      />
      {searchQuery && (
        <span className="text-xs text-muted-foreground shrink-0">
          {matchCount} found
        </span>
      )}
    </div>
  </Panel>
)}
```

**Matching logic:** Derive `matchingIds` from the full node list (both `sourceNodes` and `ontologyNodes` combined). A node matches if its label or URI contains `searchQuery` (case-insensitive). When `searchQuery` is non-empty and `matchingIds.length > 0`, call `fitView({ nodes: matchingNodes, padding: 0.3, duration: 400 })` via `useReactFlow()`.

**Highlight ring:** Pass `isHighlighted: matchingIds.includes(node.id)` (or similar) into each node's `data` when search is active. In `ClassNode` and `SourceNode`, render a highlight ring when `data.isHighlighted` is true:
```tsx
// In the node root div, add to className:
data.isHighlighted ? 'ring-2 ring-blue-400 ring-offset-1' : ''
```

**Expose openSearch:** Export a callback prop `onOpenSearch?: () => void` or use a ref/context so the keyboard shortcut hook can trigger it. A simple approach: accept `searchOpen` and `setSearchOpen` as props from the parent (App or RightPanel) that also owns the keyboard shortcut hook.

**Done:** Typing "nato" in the search bar highlights all NATO ontology class nodes and pans to them. Escape closes the bar and removes highlights.

### Task 2 — Global keyboard shortcuts hook

**File:** `src/hooks/useKeyboardShortcuts.ts` (new file)

```typescript
import { useEffect } from 'react';

interface Options {
  onOpenSearch: () => void;
  onDelete: () => void;
}

export function useKeyboardShortcuts({ onOpenSearch, onDelete }: Options) {
  useEffect(() => {
    function handle(e: KeyboardEvent) {
      // Don't fire when typing in inputs/textareas/contenteditable
      const target = e.target as HTMLElement;
      if (
        target.tagName === 'INPUT' ||
        target.tagName === 'TEXTAREA' ||
        target.isContentEditable
      ) return;

      if ((e.ctrlKey || e.metaKey) && e.key === 'f') {
        e.preventDefault();
        onOpenSearch();
        return;
      }

      if (e.key === 'Delete' || e.key === 'Backspace') {
        onDelete();
      }
    }

    document.addEventListener('keydown', handle);
    return () => document.removeEventListener('keydown', handle);
  }, [onOpenSearch, onDelete]);
}
```

**Wire in App or OntologyCanvas:** Call `useKeyboardShortcuts` at the canvas level (OntologyCanvas or its parent), passing:
- `onOpenSearch`: sets `searchOpen(true)`
- `onDelete`: calls the existing delete-selected-node/edge handler (React Flow's `onNodesDelete` / `onEdgesDelete` already handle deletion when nodes are selected and Delete is pressed natively — verify React Flow handles this automatically before adding a manual handler; if it does, the `onDelete` callback may be a no-op)

**Done:** Ctrl+F opens node search from anywhere on the canvas. Delete key removes selected nodes/edges (verify React Flow's built-in handling first).
