### Task 3 — TourProvider component

**File:** `src/components/onboarding/TourProvider.tsx` (new file)

```tsx
import Joyride, { CallBackProps, STATUS } from 'react-joyride';
import { tourSteps } from './tourSteps';

const TOUR_KEY = 'rosetta-tour-seen';

interface Props {
  running: boolean;
  onStop: () => void;
}

export function TourProvider({ running, onStop }: Props) {
  function handleCallback(data: CallBackProps) {
    const { status } = data;
    if (status === STATUS.FINISHED || status === STATUS.SKIPPED) {
      localStorage.setItem(TOUR_KEY, '1');
      onStop();
    }
  }

  return (
    <Joyride
      steps={tourSteps}
      run={running}
      continuous
      showProgress
      showSkipButton
      scrollToFirstStep
      spotlightClicks={false}
      callback={handleCallback}
      styles={{
        options: {
          primaryColor: 'hsl(var(--primary))',
          zIndex: 10000,
        },
        tooltip: {
          borderRadius: '0.5rem',
          fontSize: '0.875rem',
        },
      }}
    />
  );
}
```

**Wire into App.tsx (or main layout):**

```tsx
import { TourProvider } from '@/components/onboarding/TourProvider';

// In the App component:
const [tourRunning, setTourRunning] = useState(() => {
  return !localStorage.getItem('rosetta-tour-seen');
});

// In JSX, alongside the main layout:
<TourProvider running={tourRunning} onStop={() => setTourRunning(false)} />
```

The `useState` initializer runs once on mount — if the key is absent, tour auto-starts. If present, tour does not run.

**Done:** First-time visitor sees the tour automatically. Completing or skipping the tour sets the localStorage flag. Refreshing the page does not restart the tour.

### Task 4 — Tour button in Header

**File:** `src/components/layout/Header.tsx`

The Header needs access to `setTourRunning`. Pass it as a prop, or lift the `tourRunning` state into `uiStore` (add `tourRunning: boolean` and `setTourRunning` action).

**Simplest approach:** Add `tourRunning` / `setTourRunning` to `uiStore` (since it's already the shared UI state store). Then Header and App both read from `uiStore`.

In `uiStore.ts`, add:
```typescript
tourRunning: boolean;
setTourRunning: (v: boolean) => void;
```

Initial value: `tourRunning: !localStorage.getItem('rosetta-tour-seen')` — same check as before, but now centralized in the store.

In Header, add a Tour button (icon button with `CompassIcon` from Phosphor or similar):
```tsx
<Tooltip>
  <TooltipTrigger asChild>
    <Button
      variant="ghost"
      size="sm"
      onClick={() => setTourRunning(true)}
      aria-label="Take the tour"
    >
      <CompassIcon size={16} />
    </Button>
  </TooltipTrigger>
  <TooltipContent>Take the tour</TooltipContent>
</Tooltip>
```

Update `TourProvider` to read `tourRunning` and `setTourRunning` from `uiStore` instead of props (simplifies wiring):
```tsx
const tourRunning = useUiStore((s) => s.tourRunning);
const setTourRunning = useUiStore((s) => s.setTourRunning);
```

In App.tsx, remove the `tourRunning` useState — just render `<TourProvider />` with no props.

**Done:** A compass/tour icon button appears in the Header. Clicking it restarts the tour from step 1, even for returning visitors.
