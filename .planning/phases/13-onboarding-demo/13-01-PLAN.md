# Plan 13-01: Example Data Assets

## Tasks

### Task 1 — Expand SEED_TURTLE in ontologyStore.ts
**File:** `src/store/ontologyStore.ts`

Replace the 3-triple SEED_TURTLE with a rich ontology:

**nato:Track** (base class) — 5 shared properties:
- `nato:trackNumber` (xsd:string) — unique track identifier
- `nato:timestamp` (xsd:dateTime) — report time
- `nato:sensorId` (xsd:string) — reporting sensor identifier
- `nato:countryCode` (xsd:string) — ISO 3166-1 alpha-3 reporting nation
- `nato:trackQuality` (xsd:float) — confidence 0.0–1.0

**nato:AirTrack** (subClassOf nato:Track) — 7 air-specific properties:
- `nato:latitude` (xsd:float) — WGS-84 decimal degrees
- `nato:longitude` (xsd:float) — WGS-84 decimal degrees
- `nato:altitudeFt` (xsd:float) — altitude in feet MSL
- `nato:speedKts` (xsd:float) — speed in knots
- `nato:headingDeg` (xsd:float) — magnetic heading 0–360
- `nato:iffCode` (xsd:string) — STANAG IFF designator (FRIEND/HOSTILE/UNKNOWN/SUSPECT/ASSUMED_FRIEND/NEUTRAL)
- `nato:iffConfidence` (xsd:float) — IFF identification confidence 0.0–1.0

**Done:** `SEED_TURTLE` contains `nato:trackNumber`, `nato:altitudeFt`, `nato:iffCode`, `nato:iffConfidence`, `rdfs:label` on every class and property.

### Task 2 — Enrich Norwegian JSON source
**File:** `src/data/sample-source-a-norwegian.json`

5 radar tracks from Norwegian NASAMS/NORSE-3D radar system. Flat JSON structure (all fields at track level). Fields:
`trkNo`, `lat`, `lon`, `alt_ft`, `spd_kts`, `hdg`, `iff` (FRI/HOS/NEU/UNK), `iff_conf`, `quality`, `sensor`, `country`, `time`

Mix of IFF designations: 2 friendly, 1 hostile, 1 neutral, 1 unknown. Realistic Norwegian coordinates (58–71°N, 4–31°E).

**Done:** File has exactly 5 track objects under `radarTracks`, each with 12 fields.

### Task 3 — Enrich German JSON source
**File:** `src/data/sample-source-b-german.json`

5 radar tracks from German PATRIOT/ADLER-2000 radar. Nested JSON structure. Fields:
- `ziel_id` (string)
- `position: { breite, laenge, hoehe_m }` (nested object)
- `geschwindigkeit_kmh` (float — note: kmh, not knots)
- `kurs_grad` (float)
- `identifikation: { typ, sicherheit }` (nested object, German IFF names: FREUNDLICH/FEINDLICH/NEUTRAL/UNBEKANNT)
- `spurqualitaet` (float)
- `radar` (string)
- `land` (string)
- `zeitstempel` (string)

**Done:** File has 5 track objects under `erkannte_ziele`, each with nested `position` and `identifikation`.

### Task 4 — Create UK XML source
**File:** `src/data/sample-source-c-uk.xml` (new file)

4 radar tracks from UK Giraffe AMB/Sentinel radar in a NATO-style XML message format. Hierarchical structure:

```xml
<?xml version="1.0" encoding="UTF-8"?>
<TrackReport version="2.1" origin="GBR" classification="UNCLASSIFIED">
  <AirTrack id="UK-T0234">
    <TrackId>UK-T0234</TrackId>
    <Position>
      <Lat>51.5074</Lat>
      <Lon>-0.1278</Lon>
      <AltFt>25000</AltFt>
    </Position>
    <Kinematics>
      <SpeedKts>420</SpeedKts>
      <HeadingDeg>045</HeadingDeg>
    </Kinematics>
    <Identification>
      <IffCode>FRIEND</IffCode>
      <Confidence>0.93</Confidence>
    </Identification>
    <Quality>0.89</Quality>
    <SensorId>GIRAFFE-4A</SensorId>
    <Country>GBR</Country>
    <Timestamp>2026-04-07T09:15:00Z</Timestamp>
  </AirTrack>
  <!-- 3 more AirTrack elements -->
</TrackReport>
```

Mix of IFF codes: FRIEND, HOSTILE, UNKNOWN, ASSUMED_FRIEND. UK coordinates (50–60°N, -5–2°E).

**Done:** File parses via `xmlToSchema` and produces class nodes for `TrackReport`, `AirTrack`, `Position`, `Kinematics`, `Identification`.

### Task 5 — Enrich SHACL shapes
**File:** `src/data/sample-shapes.ttl`

Replace minimal shapes with comprehensive validation covering the enriched ontology:

- `nato:TrackShape` targeting `nato:Track`: `trackNumber` minCount 1 xsd:string, `timestamp` minCount 1 xsd:string, `sensorId` xsd:string, `countryCode` xsd:string sh:pattern `^[A-Z]{3}$`, `trackQuality` xsd:float minInclusive 0 maxInclusive 1
- `nato:AirTrackShape` targeting `nato:AirTrack`: `latitude` xsd:float minInclusive -90 maxInclusive 90, `longitude` xsd:float minInclusive -180 maxInclusive 180, `altitudeFt` xsd:float minInclusive -1500, `speedKts` xsd:float minInclusive 0, `headingDeg` xsd:float minInclusive 0 maxInclusive 360, `iffCode` xsd:string sh:pattern `^(FRIEND|HOSTILE|UNKNOWN|SUSPECT|ASSUMED_FRIEND|NEUTRAL)$`, `iffConfidence` xsd:float minInclusive 0 maxInclusive 1

**Done:** `sample-shapes.ttl` has both `nato:TrackShape` and `nato:AirTrackShape` with `sh:minInclusive`/`sh:maxInclusive` constraints.
