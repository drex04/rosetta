# Plan 13-09: Edge Context Menus

## Context

Currently, `handleEdgeDoubleClick` exists in `OntologyCanvas.tsx` (line 957) but no context menu is shown on edge interactions. The user wants double-click and right-click on any edge to open a context menu.

**Edge types in play:**
- `MappingEdge` — connects source property → ontology class. Backed by `mappingStore`. Context menu: change `kind` (direct / template / constant / typecast / language / join / formula) + Delete.
- `SubclassEdge` — `rdfs:subClassOf` relationship between ontology classes. Backed by `ontologyStore`. Context menu: "Change to Object Property" (prompts for label) + Delete.
- `ObjectPropertyEdge` — named object property edge between classes. Backed by `ontologyStore`. Context menu: "Change to Subclass" + Delete.

`ontologyStore` has `addEdge` / `removeEdge` but no `updateEdge`. That action must be added.

## Tasks

### Task 1 — Add updateEdge to ontologyStore

**File:** `src/store/ontologyStore.ts`

Add a new action after `removeEdge`:

```ts
updateEdge: (id: string, patch: Partial<Omit<OntologyEdge, 'id'>>) => void;
```

Implementation:
```ts
updateEdge: (id, patch) =>
  set((s) => ({
    edges: s.edges.map((e) => (e.id === id ? { ...e, ...patch } : e)),
  })),
```

Also export the type so `EdgeContextMenu.tsx` can import it without circular deps. The action works on the `OntologyEdge` shape already defined in the store.

**Done:** `ontologyStore.updateEdge('edge-1', { type: 'objectPropertyEdge', label: 'hasPosition' })` updates the edge in place and React Flow re-renders it.

### Task 2 — EdgeContextMenu component

**File:** `src/components/canvas/EdgeContextMenu.tsx` (new file)

```tsx
interface EdgeCtxMenu {
  edgeId: string;
  edgeType: 'mapping' | 'subclassEdge' | 'objectPropertyEdge';
  edgeData: Record<string, unknown>;
  x: number;
  y: number;
}

interface Props {
  menu: EdgeCtxMenu;
  onClose: () => void;
}
```

The component renders a `position: fixed` div at `{ left: menu.x, top: menu.y }` with `z-index: 9999`. A full-screen transparent backdrop (`position: fixed, inset: 0`) sits below the menu and calls `onClose` on click, dismissing the menu.

**Mapping edge variant** (when `menu.edgeType === 'mapping'`):

Read the current kind from `menu.edgeData.kind as string`. Render a vertical list of kind options:

```
direct | template | constant | typecast | language | join | formula
```

Each is a button. The current kind has a check icon. Clicking any kind calls:
```ts
const updateMapping = useMappingStore((s) => s.updateMapping);
updateMapping(menu.edgeId, { kind: selectedKind });
onClose();
```

Below the kind list, render a divider and a "Delete mapping" button that calls:
```ts
const removeMapping = useMappingStore((s) => s.removeMapping);
removeMapping(menu.edgeId);
onClose();
```

`removeMapping` is the existing store action. Note: the mapping id matches the edge id (convention from MappingEdge).

**Ontology edge variants:**

For `subclassEdge`:
- "Change to Object Property" button: prompts `window.prompt('Relationship name:', suggestedLabel)`. The `suggestedLabel` is derived from the edge's target node label if available (e.g. if target label is "Position", suggest "hasPosition"). If user cancels or enters empty string, do nothing. Otherwise:
  ```ts
  ontologyStore.updateEdge(menu.edgeId, { type: 'objectPropertyEdge', label: trimmedInput });
  onClose();
  ```
- "Delete edge" button: `ontologyStore.removeEdge(menu.edgeId); onClose();`

For `objectPropertyEdge`:
- "Change to Subclass" button:
  ```ts
  ontologyStore.updateEdge(menu.edgeId, { type: 'subclassEdge', label: '' });
  onClose();
  ```
- "Delete edge" button: `ontologyStore.removeEdge(menu.edgeId); onClose();`

**Styling:** `bg-background border border-border rounded-md shadow-lg py-1 min-w-[160px] text-sm`. Menu items: `px-3 py-1.5 hover:bg-muted cursor-pointer flex items-center gap-2`. Divider: `border-t border-border my-1`. Destructive action (Delete): `text-destructive hover:bg-destructive/10`.

**Done:** A correctly-typed `EdgeContextMenu` component renders both menu variants with shadcn-consistent styling.

### Task 3 — Wire context menu in OntologyCanvas

**File:** `src/components/canvas/OntologyCanvas.tsx`

Add state:
```ts
const [edgeCtxMenu, setEdgeCtxMenu] = useState<EdgeCtxMenu | null>(null);
```

Helper to derive edge type from edge id and existing edge lists:
```ts
function resolveEdgeType(edgeId: string): 'mapping' | 'subclassEdge' | 'objectPropertyEdge' {
  if (edgeId.startsWith('mapping_')) return 'mapping';
  const ontoEdge = ontologyEdges.find((e) => e.id === edgeId);
  return (ontoEdge?.type ?? 'subclassEdge') as 'subclassEdge' | 'objectPropertyEdge';
}
```

Replace the existing `handleEdgeDoubleClick` body:
```ts
const handleEdgeDoubleClick = useCallback(
  (_event: React.MouseEvent, edge: Edge) => {
    const edgeType = resolveEdgeType(edge.id);
    setEdgeCtxMenu({ edgeId: edge.id, edgeType, edgeData: edge.data ?? {}, x: _event.clientX, y: _event.clientY });
  },
  [ontologyEdges],
);
```

Add `onEdgeContextMenu` handler:
```ts
const handleEdgeContextMenu = useCallback(
  (event: React.MouseEvent, edge: Edge) => {
    event.preventDefault(); // suppress browser native menu
    const edgeType = resolveEdgeType(edge.id);
    setEdgeCtxMenu({ edgeId: edge.id, edgeType, edgeData: edge.data ?? {}, x: event.clientX, y: event.clientY });
  },
  [ontologyEdges],
);
```

Add to `<ReactFlow>`:
```tsx
onEdgeContextMenu={handleEdgeContextMenu}
```

Render `EdgeContextMenu` in the JSX (outside `<ReactFlow>`, inside the canvas wrapper so z-index stacking is correct):
```tsx
{edgeCtxMenu && (
  <EdgeContextMenu menu={edgeCtxMenu} onClose={() => setEdgeCtxMenu(null)} />
)}
```

Close on canvas click: add `onPaneClick={() => setEdgeCtxMenu(null)}` to `<ReactFlow>` (it likely already exists — add `setEdgeCtxMenu(null)` to its body).

**Done:** Double-clicking any edge shows the context menu. Right-clicking any edge shows the context menu (browser native menu suppressed). Clicking elsewhere closes it. Mapping kind changes appear instantly in the MAP tab. Ontology edge type swaps reflect on the canvas.
