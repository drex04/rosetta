Create `src/lib/rml.ts`. All exports are pure functions — NO imports from React, Zustand stores, or @xyflow/react.

**`inferIterator(jsonString: string): string`** — exported helper:
- Try `JSON.parse(jsonString)`.
- If parse fails: return `'$[*]'` (safe fallback).
- If root is an array (`Array.isArray(parsed)`): return `'$[*]'`.
- **Guard**: If `parsed` is `null` or `typeof parsed !== 'object'` (number, boolean, string): return `'$'`. This prevents `TypeError` from `Object.entries(null)`.
- If root is a plain object: find the first key whose value is an array. If found: return `'$.key[*]'` (using the actual key name). Otherwise: return `'$'` (flat object, single instance).
- Edge case: empty string → return `'$[*]'`.

**`deriveSubjectTemplate(schemaNodes: Source['schemaNodes'], classUri: string): string`** — internal helper:
- Find the schemaNode with `data.uri === classUri`.
- From `node.data.properties`, find the first property whose `localName(prop.uri)` contains `'id'` or `'key'` (case-insensitive). If found, use it as the template variable.
- If not found, use the first property.
- If no properties exist, return `'http://example.org/{classLocalName}/{index}'` as a safe IRI fallback. **Do not use blank node syntax** (`_:...`) — `rr:template` only generates IRIs.
- Template format: `'{uriPrefix}{varName}'` — e.g. `'http://src_radar_#track/{trackId}'`.

**`generateRml(sources: Source[], mappingsBySource: Record<string, Mapping[]>): string`** — main export:
- Build a Turtle string. Header:
  ```turtle
  @prefix rr: <http://www.w3.org/ns/r2rml#> .
  @prefix rml: <http://semweb.mmlab.be/ns/rml#> .
  @prefix ql: <http://semweb.mmlab.be/ns/ql#> .
  @prefix xsd: <http://www.w3.org/2001/XMLSchema#> .
  ```
- For each source in `sources`:
  - Skip if `source.json.trim() === ''`.
  - Compute `iterator = inferIterator(source.json)`.
  - Get mappings for this source: `const mlist = mappingsBySource[source.id] ?? []`.
  - Group mappings by `sourceClassUri`.
  - For each `sourceClassUri` group:
    - Derive a safe map name: `<#${localName(sourceClassUri)}Map>`.
    - Derive subject URI prefix from source name using the same pattern as jsonToSchema: `http://src_${sanitize(source.name)}_#` (sanitize: strip non-alphanum, lowercase).
    - Derive subject template: `'{uriPrefix}{classLocal}/{idProp}'` where `idProp` is the first property containing 'id' or 'key', else `'index'` as fallback.
    - Emit:
      ```turtle
      <#TrackMap> a rr:TriplesMap ;
        rml:logicalSource [
          rml:source "${source.name}.json" ;
          rml:referenceFormulation ql:JSONPath ;
          rml:iterator "${iterator}" ;
        ] ;
        rr:subjectMap [
          rr:template "${subjectTemplate}" ;
          rr:class <${sourceClassUri}> ;
        ] ;
      ```
    - For each mapping in the group:
      - If `kind === 'sparql'` or `kind === 'join'`: emit a commented-out predicate object map:
        ```turtle
          # rr:predicateObjectMap [ # requires manual conversion
          #   rr:predicate <${targetPropUri}> ;
          #   rr:objectMap [ rml:reference "..." ] ;
          # ] ;
        ```
      - Otherwise emit a real `rr:predicateObjectMap`:
        ```turtle
          rr:predicateObjectMap [
            rr:predicate <${mapping.targetPropUri}> ;
            rr:objectMap [ rml:reference "${localName(mapping.sourcePropUri)}" ] ;
          ] ;
        ```
        For `kind === 'constant'`: use `rr:object "${mapping.constantValue ?? ''}"^^<${mapping.constantType ?? 'http://www.w3.org/2001/XMLSchema#string'}>` instead of `rml:reference`.
        For `kind === 'language'`: add `rr:language "${mapping.languageTag ?? 'en'}"` to objectMap.
        For `kind === 'typecast'`: add `rr:datatype <${mapping.targetDatatype ?? 'http://www.w3.org/2001/XMLSchema#string'}>` to objectMap.
    - Close TriplesMap: ` .`
    - Separate maps with a blank line.
- Return the full Turtle string.
  </action>

**`generateYarrrml(sources: Source[], mappingsBySource: Record<string, Mapping[]>): string`**:
Build YARRRML YAML as a plain string (no js-yaml dependency — per CONTEXT.md decision).

Structure:
```yaml
prefixes:
  rr: "http://www.w3.org/ns/r2rml#"
  rml: "http://semweb.mmlab.be/ns/rml#"
  xsd: "http://www.w3.org/2001/XMLSchema#"

mappings:
  TrackMap:
    sources:
      - ["{source.name}.json~jsonpath", "{iterator}"]
    s: "{subjectTemplate}"
    po:
      - [a, "{sourceClassUri}"]
      - ["{targetPropUri}", "{sourcePropLocalName}~jsonpath]
```

Implementation rules:
- Build string line-by-line. Use 2-space indent. No trailing whitespace.
- Header block: `prefixes:` section with `rr`, `rml`, `xsd` as above.
- Empty line, then `mappings:`. Always emit the `mappings:` key, even if no sources have data — this keeps the YAML structure valid for tooling.
- For each source (skip if `source.json.trim() === ''`):
  - `iterator = inferIterator(source.json)`
  - Group mappings by `sourceClassUri` (same grouping as rml.ts).
  - For each group:
    - Map name key: `  ${localName(sourceClassUri)}Map:` (2-space indent)
    - `    sources:` (4-space) → `      - ["${source.name}.json~jsonpath", "${iterator}"]`
    - `    s: "${subjectTemplate}"` (same subject template logic as rml.ts)
    - `    po:` followed by po entries (6-space `      - `):
      - First po: `[a, "<${sourceClassUri}>"]`
      - For each mapping:
        - `kind === 'sparql'` or `kind === 'join'`: emit a YAML comment line: `      # - ["<targetPropUri>", "?"] # requires manual conversion`
        - `kind === 'constant'`: `["<${targetPropUri}>", "${value}^^<${type}>"]`
        - `kind === 'language'`: `["<${targetPropUri}>", "${localName(sourcePropUri)}~jsonpath", "lang=${tag}"]`
        - `kind === 'typecast'`: `["<${targetPropUri}>", "${localName(sourcePropUri)}~jsonpath", "datatype=<${targetDatatype}>"]`
        - Otherwise (direct/template): `["<${targetPropUri}>", "${localName(sourcePropUri)}~jsonpath"]`
- Double-quote all string values. Escape `"` in values as `\"`.
- Separate map blocks with a blank line.
- Return the full YAML string.
  </action>

**Real `ExportTab` component:**
```tsx
function ExportTab() {
  const sources = useSourcesStore((s) => s.sources)
  const mappings = useMappingStore((s) => s.mappings)

  const hasMappings = Object.values(mappings).some((m) => m.length > 0)
  const hasSparqlOrJoin = Object.values(mappings).flat()
    .some((m) => m.kind === 'sparql' || m.kind === 'join')

  function handleDownloadRml() {
    const rml = generateRml(sources, mappings)
    downloadBlob('mappings.rml.ttl', rml, 'text/turtle')
  }

  function handleDownloadYarrrml() {
    const yaml = generateYarrrml(sources, mappings)
    downloadBlob('mappings.yarrrml.yml', yaml, 'application/yaml')
  }

  if (!hasMappings) {
    return (
      <div className="p-4 text-xs text-muted-foreground">
        No mappings defined yet. Create mappings in the MAP tab to export RML/YARRRML.
      </div>
    )
  }

  return (
    <div className="flex flex-col gap-3 p-3">
      {hasSparqlOrJoin && (
        <div className="flex items-start gap-2 p-2.5 rounded bg-amber-50 border border-amber-200 text-xs text-amber-800">
          <WarningIcon size={14} className="shrink-0 mt-0.5" />
          <span>
            Some mappings use <strong>SPARQL</strong> or <strong>join</strong> kinds and are annotated
            as <em>requires manual conversion</em> in the exported files.
          </span>
        </div>
      )}
      <div className="flex flex-col gap-2">
        <p className="text-xs text-muted-foreground">
          Download production ETL mapping files for use with RML processors (RMLMapper, Morph-KGC, etc.).
        </p>
        <Button
          variant="outline"
          size="sm"
          className="justify-start gap-2 text-xs"
          onClick={handleDownloadRml}
        >
          <DownloadSimpleIcon size={14} />
          Download RML (.rml.ttl)
        </Button>
        <Button
          variant="outline"
          size="sm"
          className="justify-start gap-2 text-xs"
          onClick={handleDownloadYarrrml}
        >
          <DownloadSimpleIcon size={14} />
          Download YARRRML (.yarrrml.yml)
        </Button>
      </div>
    </div>
  )
}
```

Note: `useMappingStore` and `useSourcesStore` should already be imported from Plan 06-02. Check before adding — do not duplicate. If missing, add: `import { useMappingStore } from '@/store/mappingStore'` and `import { useSourcesStore } from '@/store/sourcesStore'`.
