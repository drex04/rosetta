Install Comunica and build the data fusion pipeline as a pure library: execute each source's SPARQL CONSTRUCT queries, merge output quads into a unified N3.Store, annotate provenance, then compact to JSON-LD. Wire the pipeline into an ephemeral fusionStore.
</objective>

<task type="auto">
  <name>Task 1: Install Comunica and create fusion.ts</name>
  <files>src/lib/fusion.ts</files>
  <action>
1. Run `npm install @comunica/query-sparql`.
2. Create `src/lib/fusion.ts`. Comunica MUST be lazy-loaded via dynamic import to defer the ~4 MB chunk from initial load:
   ```ts
   let _engine: import('@comunica/query-sparql').QueryEngine | null = null
   async function getEngine() {
     if (!_engine) {
       const { QueryEngine } = await import('@comunica/query-sparql')
       _engine = new QueryEngine()
     }
     return _engine
   }
   ```
3. Export `FusionSourceResult` interface: `{ sourceId: string; sourceName: string; quadCount: number }`.
4. Export `FusionResult` interface: `{ store: N3.Store; sources: FusionSourceResult[]; totalQuads: number }`.
5. Export `async function executeAllConstructs(sources: Source[], mappingsBySource: Record<string, Mapping[]>, ontologyNodes: OntologyNode[]): Promise<FusionResult>`:
   - Create a merged `N3.Store`.
   - `const engine = await getEngine()`.
   - For each source in `sources`:
     - Skip if `source.json.trim() === ''` or no mappings for source.
     - Parse `source.json` with `JSON.parse`; wrap in try/catch (skip source on invalid JSON).
     - Collect the source's CONSTRUCT queries: use `sparqlConstruct` from each mapping for `kind === 'sparql'` or `kind === 'direct'`; for other kinds, skip (they will be handled when kind expansion lands in Plan 06-02).
     - For each CONSTRUCT query:
       - Build an in-memory source store: parse `source.json` via jsonToInstances (import from `src/lib/shacl/instanceGenerator.ts`).
       - Run query via Comunica: `const quadStream = await engine.queryQuads(sparqlQuery, { sources: [instanceStore] })`.
       - Collect quads: `const quads: N3.Quad[] = await quadStream.toArray()`.
       - Add all quads to merged store.
     - Record per-source quad count before provenance annotation.
   - **Provenance annotation**: After collecting all quads, for each source iterate over `mergedStore.getSubjects(null, null, null)` — for each distinct subject add:
     `<subject> <http://www.w3.org/ns/prov#wasAttributedTo> "<sourceName>"^^xsd:string`
     using `N3.DataFactory.literal(source.name)`.
     Only add provenance for subjects that were contributed by this source (track per-source subject sets during collection).
   - Return `{ store: mergedStore, sources: sourceSummaries, totalQuads: mergedStore.size }`.
6. Guard: if `sources` is empty or all are skipped, return `{ store: new N3.Store(), sources: [], totalQuads: 0 }`.
  </action>

<task type="auto">
  <name>Task 2: jsonldFramer.ts — N3.Store → JSON-LD compact</name>
  <files>src/lib/jsonldFramer.ts</files>
  <action>
1. Create `src/lib/jsonldFramer.ts`.
2. Export `async function compactToJsonLd(store: N3.Store, ontologyNodes: OntologyNode[]): Promise<object>`:
   - Serialize `store` to N-Quads using `N3.Writer` with `format: 'N-Quads'`.
   - Convert to JSON-LD dataset: `const dataset = await jsonld.fromRDF(nquads, { format: 'application/n-quads' })`.
   - Build context object from `ontologyNodes`: for each node, derive a short prefix key from `localName(node.data.uri)` (lowercase) mapped to the node's URI prefix (strip trailing `#` or `/` character). Deduplicate by prefix key (first wins).
     Example: node with URI `http://nato.int/air#Track` → `{ "track": "http://nato.int/air#" }`.
   - Add standard prefixes: `{ "prov": "http://www.w3.org/ns/prov#", "xsd": "http://www.w3.org/2001/XMLSchema#" }`.
   - `const compacted = await jsonld.compact(dataset, context)`.
   - Return `compacted`.
3. Guard: if `store.size === 0`, return `{ "@context": {}, "@graph": [] }` directly (skip jsonld calls).
4. Wrap the entire operation in try/catch; rethrow with prefix `"JSON-LD compaction failed: "`.
  </action>

  <name>Task 4: fusionStore.ts — ephemeral Zustand store</name>
  <files>src/store/fusionStore.ts</files>
  <action>
1. Create `src/store/fusionStore.ts`.
2. Define `FusionState` interface:
   ```ts
   interface FusionState {
     result: FusionResult | null
     jsonLd: object | null
     loading: boolean
     stale: boolean
     error: string | null
     lastRun: number | null
     runFusion: () => Promise<void>
     setStale: (stale: boolean) => void
     reset: () => void
   }
   ```
3. Implement `useFusionStore = create<FusionState>()(...)`:
   - `runFusion`:
     1. If `loading` return early.
     2. `set({ loading: true, error: null })`.
     3. Read sources from `useSourcesStore.getState()`, mappings from `useMappingStore.getState()`, ontologyNodes from `useOntologyStore.getState()`.
     4. `const fusionResult = await executeAllConstructs(sources, mappings.mappings, ontologyNodes.nodes)`.
     5. `const jsonLd = await compactToJsonLd(fusionResult.store, ontologyNodes.nodes)`.
     6. `set({ result: fusionResult, jsonLd, loading: false, stale: false, lastRun: Date.now() })`.
     7. Catch: `set({ loading: false, error: err.message ?? 'Fusion failed' })`.
   - `setStale`: `set({ stale })`.
   - `reset`: reset all fields to initial state.
4. **No IDB persistence** — per locked decision, results are ephemeral.
5. Add `setStale(true)` subscription: subscribe to mappingStore and sourcesStore changes (check via `useEffect` in the consumer, not in the store itself — note this for the UI wiring in Plan 06-02).
