**Tests to write first (src/__tests__/jsonToSchema.test.ts):**
- Primitive fields → DatatypeProperty (string→xsd:string, number with decimal→xsd:float, integer→xsd:integer, boolean→xsd:boolean)
- Array root → Class (PascalCase of key as-is, NO singularization: 'radarTracks' → 'RadarTracks')
- Nested object → separate Class + ObjectProperty link
- Circular reference → does not throw; circular-referencing property suppressed; path string in warnings[]; parent Class still emitted
- Empty input `{}` or `[]` → returns `{ nodes: [], edges: [], turtle: '', warnings: [] }`
- Null root (`JSON.parse('null')`) → returns `{ nodes: [], edges: [], turtle: '', warnings: ['Unexpected root type'] }`
- Primitive root (`JSON.parse('"hello"')`) → returns `{ nodes: [], edges: [], turtle: '', warnings: ['Unexpected root type'] }`
- Source name with special chars ('Norway/Track#Alpha') → URI prefix is sanitized to 'src_norwayTrackalpha_'
- Root object with array property (`{ radarTracks: [...] }`) → class name = 'RadarTracks' (no singularization) with primitive properties
- N3.Writer failure: mock N3.Writer to throw → jsonToSchema returns `{ nodes, edges, turtle: '', warnings: ['Failed to serialize schema to Turtle'] }`

**Implementation (src/lib/jsonToSchema.ts):**

```typescript
export interface SchemaResult {
  nodes: Node[]       // ReactFlow nodes, type='sourceNode'
  edges: Edge[]       // ReactFlow edges
  turtle: string      // Generated RDFS Turtle
  warnings: string[]  // e.g. circular ref paths
}

export function jsonToSchema(json: string, sourceName: string): SchemaResult
```

Algorithm:
1. JSON.parse — catch SyntaxError, return empty result with `warnings: ['Invalid JSON']`. After parse: if result is null, a string, number, or boolean, return `{ nodes: [], edges: [], turtle: '', warnings: ['Unexpected root type'] }` immediately.
2. Derive URI prefix from sourceName: replace all non-alphanumeric chars (except underscore) with empty string, then lowercase, prefix with `src_`, suffix with `_`. e.g. `'Norwegian Radar'` → `'src_norwegianradar_'`. This sanitization prevents IRI injection from special characters in source names.
3. Walk the value recursively. Track visited objects via WeakSet for circular detection.
4. Array root: take first element as representative, class name = PascalCase(key) with NO singularization (language-agnostic; 'radarTracks' → 'RadarTracks', 'Radarspuren' → 'Radarspuren')
5. Object properties:
   - Primitive → DatatypeProperty, XSD type from value type
   - Object → recurse, emit new Class + ObjectProperty
   - Array of primitives → DatatypeProperty (xsd:string for mixed)
   - Array of objects → recurse on first element, same as object branch
6. Assign column positions: x = COLUMN_X_SOURCE, y = classIndex * COLUMN_SPACING (import from rdf.ts)
7. Serialize classes/properties to N3 quads → Turtle string via N3.Writer. Wrap N3 serialization in try/catch: on any N3 error, return `{ nodes, edges, turtle: '', warnings: [...existingWarnings, 'Failed to serialize schema to Turtle'] }`

   - Use `isUpdatingFromStore` ref flag to prevent circular updates when store changes externally. IMPORTANT: unlike TurtleEditorPanel, do NOT add a `hasFocus` guard — source switching must always update the editor content even if it currently has focus. Do NOT read activeSourceId from the store inside the debounce callback — the active source may have switched before the 500ms fires, causing a cross-source write.
   - Below editor: collapsible "Generated RDFS" section (collapsed by default). When open: read-only CodeMirror with turtle language mode, content = last successful result.turtle.
   - Prefix collision check: after a successful jsonToSchema call, derive the slug from source.name and compare against all other sources' name slugs.
