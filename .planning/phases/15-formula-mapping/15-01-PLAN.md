<task type="auto" tdd="true">
  <name>Task 2: formulaParser.ts — tokenizer and recursive descent parser</name>
  <files>src/lib/formulaParser.ts, src/__tests__/formulaParser.test.ts</files>
  <action>
Create `src/lib/formulaParser.ts`. No external dependencies.

**AST types:**
```typescript
export type Expr =
  | { type: 'call'; fn: string; args: Expr[] }
  | { type: 'field'; path: string }   // source.fieldName
  | { type: 'literal'; value: string }

export type ParseResult = { ast: Expr; errors: string[] }
```

**Tokenizer** — produces tokens: IDENT, DOT, LPAREN, RPAREN, COMMA, STRING, NUMBER, EOF.

**Parser** — recursive descent:
- `parseExpr()` → tries `parseCall()`, then `parseFieldRef()`, then `parseLiteral()`
- `parseCall()` → IDENT LPAREN [parseExpr (COMMA parseExpr)*] RPAREN
- `parseFieldRef()` → `source` DOT IDENT
- `parseLiteral()` → STRING | NUMBER

**Whitelist validator** — after parsing, walk AST and validate:

```typescript
const ALLOWED_FUNCTIONS: Record<string, { minArgs: number; maxArgs: number }> = {
  CONCAT:  { minArgs: 2, maxArgs: 8 },
  UPPER:   { minArgs: 1, maxArgs: 1 },
  LOWER:   { minArgs: 1, maxArgs: 1 },
  TRIM:    { minArgs: 1, maxArgs: 1 },
  REPLACE: { minArgs: 3, maxArgs: 3 },
};
```

Unknown function name → error. Wrong arity → error.

**Exports:**
```typescript
export function parseFormula(input: string): Expr   // throws on parse error
export function validateFormula(expr: Expr): string[]  // returns error strings
export function parseAndValidate(input: string): ParseResult  // never throws
```

Write tests in `src/__tests__/formulaParser.test.ts`:
- Happy: CONCAT(source.a, source.b), UPPER(source.name), REPLACE(source.id, "-", "_")
- Nested: CONCAT(UPPER(source.a), source.b)
- Errors: unknown function FOO(), wrong arity UPPER(a, b), missing closing paren
- Field ref: source.myField parsed as { type: 'field', path: 'myField' }
- Literal: "hello" parsed as { type: 'literal', value: 'hello' }
- Empty string: `parseFormula('')` throws (or `parseAndValidate('')` returns non-empty errors — document which and be consistent)
  </action>

**rml.ts — implement `emitFnOPOM()`:**

Uses legacy fnml: syntax. Prefixes to add to the header:
```
@prefix fno: <https://w3id.org/function/ontology#> .
@prefix fnml: <http://semweb.mmlab.be/ns/fnml#> .
@prefix grel: <http://users.ugent.be/~bjdmeest/function/grel.ttl#> .
```

For a formula mapping, replace the plain `rr:predicateObjectMap` block with:
```turtle
rr:predicateObjectMap [
  rr:predicate <{targetPredicate}> ;
  rr:objectMap [
    fnml:functionValue [
      rml:logicalSource :{sourceName}_source ;
      rr:subjectMap [ rr:termType rr:BlankNode ] ;
      rr:predicateObjectMap [
        rr:predicate fno:executes ;
        rr:objectMap [ rr:constant grel:{fnName} ]
      ] ;
      {parameter predicateObjectMaps...}
    ]
  ]
] ;
```

**Function → FnO mapping:**
| Formula fn | grel: URI | Param predicates |
|---|---|---|
| UPPER | grel:toUpperCase | grel:valueParameter |
| LOWER | grel:toLowerCase | grel:valueParameter |
| TRIM | grel:trim | grel:valueParameter |
| REPLACE | grel:string_replace | grel:valueParameter, grel:param2Str (find), grel:param3Str (replace) |
| CONCAT | grel:string_concat | grel:valueParameter, grel:valueParameter2 |

For CONCAT with 3+ args, emit a chain: inner concat first as a nested fnml:functionValue blank node, outer concat references it.

For field refs (`source.fieldName`): emit `rml:reference "fieldName"`.
For string literals: emit `rr:constant "value"`.

Blank node IDs: use a counter scoped to the current mapping (e.g., `_:fn_{mappingId}_0`).

**Call site in generateRml():** In the existing `generateRml()` function, inside the loop that emits `rr:predicateObjectMap` blocks for each mapping, add a branch for `kind === 'formula'`:
```typescript
if (m.kind === 'formula') {
  try {
    const ast = parseFormula(m.formulaExpression ?? '');
    output += emitFnOPOM(ast, m, sourceName, counter);
  } catch (e) {
    output += `  # formula-error: ${(e as Error).message}\n`;
  }
}
```
`emitFnOPOM(ast: Expr, mapping: Mapping, sourceName: string, counter: { n: number }): string` — counter is a shared object so blank node IDs stay unique across multiple formula mappings in one document.

**String literal escaping:** Before inserting any `{ type: 'literal' }` value into Turtle output, escape it:
```typescript
function turtleEscape(s: string): string {
  return s.replace(/\\/g, '\\\\').replace(/"/g, '\\"');
}
```
Use `rr:constant "${turtleEscape(expr.value)}"` for all literal nodes.
