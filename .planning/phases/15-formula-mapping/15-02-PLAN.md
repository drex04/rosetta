<task type="auto" tdd="true">
  <name>Task 2: FormBuilder component (Tier 1)</name>
  <files>src/components/panels/MappingPanel.tsx</files>
  <action>
Add a `FormBuilder` sub-component inside MappingPanel.tsx (or as a local function). Renders when `kind === 'formula'` and the active tier is `'form'`.

**Structure:**
```
Function: [CONCAT ▾]           ← <select> with CONCAT/UPPER/LOWER/TRIM/REPLACE
Arg 1: [source.firstName   ]   ← <input text>
Arg 2: [" "                ]
Arg 3: [source.lastName    ]   [+ Add arg] [× Remove]  ← only shown when fn supports variadic
```

**Behaviour:**
- On mount / when formulaExpression changes externally: attempt `parseAndValidate(formulaExpression)`. If result is a single `call` AST node with a recognised function, populate the function picker and arg fields from the AST.
- On any field change: reconstruct the formula string and call `updateMapping(id, { formulaExpression: newExpr })`.
- Formula string reconstruction: `FN(arg1, arg2, ...)` where arg is either the raw input text (for field refs like `source.x`) or a quoted string (for literals).
- Arity limits: UPPER/LOWER/TRIM → 1 arg (fixed); REPLACE → 3 args (fixed); CONCAT → 2–8 args (add/remove buttons).
- Style: use `text-sm` inputs with `border border-border rounded px-2 py-1`, consistent with existing MAP tab styling. Use `size="sm"` shadcn Button for Add/Remove.
  </action>
  <verify>
    - Selecting CONCAT in function picker and filling 2 arg fields stores `CONCAT(arg1, arg2)` in formulaExpression.
    - Clicking "Add arg" appends a new empty input field; expression updates to `CONCAT(a, b, )` (empty arg).
    - Switching function from CONCAT to UPPER removes extra arg fields.
  </verify>
  <done>FormBuilder renders for formula kind; form changes update formulaExpression in store; arity-appropriate fields shown per function.</done>
</task>

<task type="auto" tdd="true">
  <name>Task 3: FormulaBar component (Tier 2) and tier toggle</name>
  <files>src/components/panels/MappingPanel.tsx</files>
  <action>
Add tier state: `const [formulaTier, setFormulaTier] = useState<'form' | 'formula' | 'rml'>('form')` — local to the formula kind area, reset when a different mapping is selected.

**Tier reset mechanism:** Wrap the tier toggle + tier content area in a container that uses the selected mapping's ID as a React key:
```tsx
<div key={selectedMapping?.id}>
  {/* tier toggle buttons */}
  {/* tier content (FormBuilder / FormulaBar / RML pane) */}
</div>
```
This unmounts and remounts the component (resetting all local state to defaults) whenever the selection changes, without needing a useEffect.

**Tier toggle UI** — rendered only when `kind === 'formula'`:
```
[Form] [Formula] [RML]
```
Three small buttons. Active tier has `bg-muted` background. Use `size="sm"` shadcn Button or plain `<button>` styled consistently with the tab pattern.

**FormulaBar sub-component** — renders when tier is `'formula'`:
- A single-line `<input type="text">` (not CodeMirror for plan 02 — CodeMirror custom language extension deferred to avoid scope creep; plain input is sufficient for plan 02).
- Value bound to `selectedMapping.formulaExpression ?? ''`.
- On change: `updateMapping(id, { formulaExpression: value })`.
- Validation badge (inline after the input): call `parseAndValidate(value)`. If `errors.length === 0` show green "valid" badge (same style as existing mapping badges: `bg-mapping/15 text-mapping-text`). If errors, show red badge with the first error message (`bg-source/15 text-source-text`).
- Empty string → no badge (neutral state).

**Bidirectional sync (Form ↔ Formula):**
- When switching from Formula → Form: if `parseAndValidate(formulaExpression)` returns a single call with no errors, the FormBuilder reads from the AST. If it has errors, FormBuilder shows the expression as a read-only text note: "Expression too complex for form view."
- When switching from Form → Formula: the formula bar shows the current `formulaExpression` from the store (already updated by FormBuilder on each change).

**RML tier:**
- When tier is `'rml'`: render the existing RML snippet pane (already shows FnO Turtle from plan 01). No new component needed — just show/hide the existing pane.

**Group formula support:** If the panel is showing a `MappingGroup` (rather than a `Mapping`), and the group has a `formulaExpression`, display a simplified 2-tier toggle (Formula / RML only — no Form tier for groups in plan 02). The FormulaBar component is reused as-is; it calls `updateGroup(id, { formulaExpression: value })` instead of `updateMapping`.
  </action>
  <verify>

    - Clicking Formula tab shows a text input with the current formulaExpression.
    - Typing `UPPER(source.name)` shows green "valid" badge.
    - Typing `FOO(source.x)` shows red badge with error text.
    - Clicking RML tab shows the RML snippet pane.
    - Selecting a different mapping resets the active tier back to Form.
  </verify>
  <done>Tier toggle works; FormulaBar shows validation badge; bidirectional sync between form and formula tiers; RML tab shows FnO Turtle; tier resets on mapping change.</done>
