  <action>
  1. Create `src/hooks/useSourceSync.ts` mirroring `useOntologySync`:
     - Ref guards: `isUpdatingFromCanvas`, `isUpdatingFromEditor` (useRef, NOT useState)
     - `onSourceEditorChange(newTurtle: string)`: immediately write to store via `updateSource(activeSourceId, { turtleSource })`, debounce 600ms, then parse → convert to source nodes → update store. On error: set parseError.
     - `onSourceCanvasChange(nodes: SourceNode[], edges: OntologyEdge[])`: if not updating from editor, serialize via `sourceCanvasToTurtle`, write to store.
     - **Critical:** On debounce timer fire, re-read `activeSourceId` from `useSourcesStore.getState()`. If it differs from the ID captured at debounce start, discard and return. Also clear pending timers when `activeSourceId` changes (useEffect cleanup).
  2. Return `{ onSourceEditorChange, onSourceCanvasChange, resetSourceSchema }`.
  3. `resetSourceSchema()`: re-read `rawData` + `dataFormat` from active source, run `jsonToSchema`/`xmlToSchema`, replace schemaNodes/schemaEdges/turtleSource, clear parseError.
  4. Write tests: (a) editor change → nodes update after debounce, (b) canvas change → turtle updates, (c) source switch during debounce → discards stale parse, (d) reset re-generates from rawData.
  </action>
  <verify>`npm run test` — useSourceSync tests pass. No circular update loops.</verify>
  <done>Bidirectional source sync works with circular-update prevention and source-switch safety.</done>
