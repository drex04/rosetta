     export interface MappingGroup {
       id: string
       strategy: 'concat' | 'coalesce' | 'template'

     - `updateGroup(groupId: string, patch: Partial<MappingGroup>): void` — update strategy/separator/template, regenerate SPARQL
     - `ungroupMappings(groupId: string): void` — clear groupId from all member mappings, remove group, regenerate individual SPARQL for each mapping
     - `getGroupsForSource(sourceId: string): MappingGroup[]`
     - `getMappingsInGroup(groupId: string): Mapping[]`
  4. Update `hydrate` to restore groups from IDB. Update `reset` to clear groups.
  5. Write tests: createGroup sets groupId on members, updateGroup changes strategy, ungroupMappings restores individual mappings, duplicate target detection works.
  </action>

<task type="tdd">
  <name>Task 2: Group SPARQL generation</name>
  <files>src/lib/sparql.ts, src/__tests__/mappingGroups.test.ts</files>
  <action>
  1. Add `generateGroupConstruct(group: MappingGroup, members: Mapping[]): string` to sparql.ts.
  2. Strategy implementations:
     - `concat`: ORDER members by `groupOrder`. Generate:
       ```sparql
       CONSTRUCT { ?target a tgt:Class . ?target tgt:prop ?joinedVal . }
       WHERE {
         ?source a src:Class .
         ?source src:prop0 ?v0 . ?source src:prop1 ?v1 .
         BIND(CONCAT(STR(?v0), "{separator}", STR(?v1)) AS ?joinedVal)
       }
       ```
     - `coalesce`: Similar but `BIND(COALESCE(?v0, ?v1, ...) AS ?joinedVal)` with OPTIONAL for each source prop.
     - `template`: Parse `templatePattern`, replace `{0}`, `{1}` etc. with `STR(?vN)` inside CONCAT.
  3. Wire `generateGroupConstruct` into `createGroup` and `updateGroup` — auto-generate and store `sparqlConstruct` on the group.
  4. Write tests: concat with 2 fields produces valid SPARQL, coalesce uses OPTIONAL + COALESCE, template substitution works, ordering respected.
  </action>
  <verify>`npm run test` — SPARQL generation tests pass. Generated queries are valid CONSTRUCT syntax.</verify>
  <done>Grouped mappings generate correct SPARQL for all three strategies.</done>
</task>
