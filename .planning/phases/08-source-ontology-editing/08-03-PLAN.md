  <name>Task 1: Granular ontologyStore actions</name>
  <files>src/store/ontologyStore.ts, src/types/index.ts, src/__tests__/ontologyCanvasEditor.test.ts</files>
  <action>
  1. Add granular actions to ontologyStore (currently only has bulk `setNodes`/`setEdges`):
     - `addNode(node: OntologyNode): void` — append to nodes array
     - `removeNode(nodeId: string): void` — remove from nodes, remove all edges referencing nodeId, trigger mapping invalidation for properties on that node
     - `addPropertyToNode(nodeId: string, property: PropertyData): void` — add to node.data.properties array
     - `removePropertyFromNode(nodeId: string, propertyUri: string): void` — remove from properties, trigger mapping invalidation
     - `addEdge(edge: OntologyEdge): void` — append to edges array
     - `removeEdge(edgeId: string): void` — remove from edges
  2. Each mutation should trigger the canvas→editor sync path (the existing `onCanvasChange` callback will handle this since nodes/edges change).
  3. Write tests: addNode increases node count, removeNode cascades to edges and mappings, addProperty/removeProperty update node data, addEdge/removeEdge work correctly.
  </action>
  <verify>`npm run test` — ontologyCanvasEditor tests pass. `npm run build` — no TS errors.</verify>
  <done>OntologyStore has granular CRUD actions that cascade correctly.</done>
</task>

<task type="auto">
  <name>Task 2: Canvas and node context menus</name>
  <files>src/components/canvas/CanvasContextMenu.tsx, src/components/canvas/NodeContextMenu.tsx, src/components/canvas/AddPropertyDialog.tsx, src/components/canvas/OntologyCanvas.tsx</files>
  <action>
  1. Create `CanvasContextMenu.tsx` — a shadcn ContextMenu or DropdownMenu that appears on right-click of the canvas pane (not on a node). Options:
     - "Add Class" — calls `ontologyStore.addNode()` with a new ClassNode at the click position, default name "NewClass", empty properties. Generate a unique URI.
  2. Create `NodeContextMenu.tsx` — appears on right-click of a ClassNode. Options:
     - "Add Property" — opens `AddPropertyDialog` (name, URI, XSD type picker)
     - "Delete Node" — calls `ontologyStore.removeNode(nodeId)` with confirmation if node has mappings
     - "Rename" — inline edit of class name
  3. Create `AddPropertyDialog.tsx` — shadcn Dialog with inputs for property name, property URI (auto-generated from name), XSD datatype picker (dropdown of common types). On submit, calls `ontologyStore.addPropertyToNode()`.
  4. Wire context menus in OntologyCanvas.tsx:
     - `onPaneContextMenu` → show CanvasContextMenu at cursor position
     - Pass `onContextMenu` handler to ClassNode via node data or wrapper
  5. For source nodes: add the same context menu pattern but route actions through `sourcesStore.updateSource()` instead of `ontologyStore`. Reuse the dialog components with a `target: 'ontology' | 'source'` prop.
  </action>
  <verify>`npm run build`. In browser: right-click canvas → "Add Class" creates a blue node. Right-click node → "Add Property" opens dialog. "Delete Node" removes it.</verify>
  <done>Context menus enable creating/deleting nodes and properties from the canvas.</done>
</task>
