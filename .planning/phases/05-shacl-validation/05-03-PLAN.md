```tsx
// Source-scoped: shows violations for the active source only.
```
Read `activeSourceId` from `useSourcesStore`. Read `{ results, stale, loading, error }` from `useValidationStore`. Read `runValidation` from `useValidationStore`.

States to render:
- **No active source**: `<p className="text-sm text-muted-foreground p-4">Select a source to validate.</p>`
- **Error**: `<Alert variant="destructive">` with `error` message.
- **Not yet run** (results[activeSourceId] is undefined, no error, not loading): `<p className="text-sm text-muted-foreground p-4">Click Validate to run.</p>`
- **Stale banner** (stale === true AND results exist): amber `<Alert>` at top: `"Mappings changed — re-validate to refresh"`. Keep the violation list visible below it.
- **Loading**: spinner centered in panel.
- **Valid** (results[activeSourceId] is empty array): `<p className="text-sm text-green-600 p-4">✓ All valid</p>`.
- **Violations**: list of `ViolationRecord`. For each:
  - Show `targetClassUri` local name + `targetPropUri` local name (use `localName` from `@/lib/rdf`).
  - Show `message` truncated to 2 lines.
  - `cursor-pointer` if `canvasNodeId !== null`, `cursor-default` otherwise.
  - `onClick`: if `canvasNodeId !== null`, call `setHighlightedCanvasNodeId(canvasNodeId)` from validationStore, and call `setActiveSourceId(violation.sourceId)` from sourcesStore.

**SourceStatusBadge (inline in SourceSelector.tsx, not a separate file):**
Add a small badge to each source pill in `SourceSelector.tsx`:
- Use **two separate Zustand selectors** (not an inline object) to avoid spurious re-renders:
  ```ts
  const results = useValidationStore((s) => s.results)
  const lastRun = useValidationStore((s) => s.lastRun)
  ```
- Do NOT import from `useMappingStore` — badge status is derived solely from `results` and `lastRun`.
- For each source, compute status:
  - If `lastRun === null`: `○` (never validated) — muted color.
  - Else if `results[source.id]` is undefined: `○` — muted color.
  - Else if `results[source.id].length === 0`: `✓` — green.
  - Else: `⚠` — amber.
- Render as a `<span>` with `text-[10px]` immediately after the source name button and before the delete button.
- Colors: `✓` = `text-green-500`, `⚠` = `text-amber-500`, `○` = `text-muted-foreground/50`.

**validationStore.ts additions:**

- Update `runValidation`'s final `set()` call (step 5) to include `highlightedCanvasNodeId: null` — the stub currently sets `{ loading: false, stale: false, lastRun: Date.now() }`; the full implementation added in 05-02 must also clear the highlight. Confirm this field is included in that final set.

**SourceNode.tsx:**
Add a `useEffect`-free approach: use a **boolean Zustand selector** to avoid re-rendering all SourceNode instances on every highlight change. Apply a ring class to the node's outer `<div>`:
```tsx
// Boolean selector: only this node re-renders when its own highlighted state changes
const isHighlighted = useValidationStore((s) => s.highlightedCanvasNodeId === id)
// ...
<div className={[
  'bg-white border-2 rounded-md shadow-md min-w-[200px] ...',
  isHighlighted ? 'ring-2 ring-destructive ring-offset-2' : '',
].join(' ')}>
```
Note: `id` is available as `props.id` on `NodeProps` — destructure it alongside `data`.

**OntologyCanvas.tsx:**
Add a `useEffect` that watches `highlightedCanvasNodeId`:
```ts
const highlightedCanvasNodeId = useValidationStore((s) => s.highlightedCanvasNodeId)

useEffect(() => {
  if (!highlightedCanvasNodeId || !rfInstance.current) return
  rfInstance.current.fitView({ padding: 0.4, duration: 400 })
}, [highlightedCanvasNodeId])
```
Note: `rfInstance.current.fitView` does not support per-node scoping in the current codebase ref type — calling it without nodes fits all visible nodes. This is acceptable for Phase 5; it scrolls the canvas to show the highlighted node in context.
