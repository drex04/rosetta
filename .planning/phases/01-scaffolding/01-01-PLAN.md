  <files>package.json, tsconfig.json, tsconfig.node.json, vite.config.ts, index.html, src/main.tsx, src/App.tsx</files>
  <action>
1. Scaffold with: `npm create vite@latest . -- --template react-ts --yes` (in the existing project root — `--yes` suppresses the "directory not empty" interactive prompt)
2. Set `strict: true` in tsconfig.json (and `noUncheckedIndexedAccess: true`)
3. Add `base: './'` to vite.config.ts for portable static export
4. Install all Phase 1–2 deps in one pass to avoid repeated installs:
   `npm install @xyflow/react zustand @radix-ui/react-tabs @radix-ui/react-tooltip @radix-ui/react-dialog @radix-ui/react-slot class-variance-authority clsx tailwind-merge`
5. Install Tailwind and toolchain:
   `npm install -D tailwindcss postcss autoprefixer @tailwindcss/typography`
   `npx tailwindcss init -p`
6. Replace `src/App.tsx` with a minimal placeholder that renders `<h1>Rosetta</h1>` inside a `<div className="min-h-screen bg-background">`.
  </action>
  <verify>
- `npm run dev` serves without errors
- `tsconfig.json` contains `"strict": true`

  </verify>

<task type="auto">
  <name>Task 2: Configure Tailwind, shadcn/ui, and semantic color system</name>
  <files>tailwind.config.ts, postcss.config.js, src/index.css, components.json</files>
  <action>
1. Run shadcn init: `npx shadcn@latest init --preset bcivVKZU`
   - If prompted for components.json options, accept defaults except: set icon library to Phosphor.
   - The preset may or may not configure Phosphor correctly.
2. Verify `components.json` references Phosphor, not Lucide. If `"iconLibrary": "lucide"` appears (or is missing), manually edit it to `"iconLibrary": "phosphor"`.
3. Install Phosphor Icons npm package: `npm install @phosphor-icons/react`
   (Note: do NOT use `npx shadcn@latest add phosphor-icons` — that is not a valid shadcn component and will fail.)
4. Add semantic color CSS custom properties to `src/index.css` in the `:root` block:
   ```css
   --color-source: 38 92% 50%;        /* amber — source schema nodes */
   --color-master: 221 83% 53%;       /* blue/indigo — master ontology nodes */
   --color-mapping: 142 71% 45%;      /* green — mapping edges */
   --color-source-fg: 0 0% 100%;
   --color-master-fg: 0 0% 100%;
   ```
5. Extend `tailwind.config.ts` colors to expose semantic classes:
   ```ts
   colors: {
     source: 'hsl(var(--color-source) / <alpha-value>)',
     master: 'hsl(var(--color-master) / <alpha-value>)',
     mapping: 'hsl(var(--color-mapping) / <alpha-value>)',
   }
   ```
6. Add a shadcn Button component to verify the setup: `npx shadcn@latest add button`
7. Render the Button in `src/App.tsx` temporarily to confirm it imports and renders.
  </action>
  <verify>
- `components.json` exists and does not contain `"lucide"` as icon library
- `src/index.css` contains `--color-source`
- `tailwind.config.ts` contains `color-source` extension
- `npm run dev` renders a shadcn Button without errors
  </verify>
  <done>shadcn/ui initialised with preset bcivVKZU; Phosphor Icons confirmed; semantic CSS vars defined; Button renders in dev server.</done>
</task>

<task type="tdd">
  <name>Task 3: Configure Vitest</name>
  <files>vitest.config.ts, src/__tests__/setup.ts, src/__tests__/smoke.test.ts</files>
  <action>
1. Install test deps: `npm install -D vitest @vitest/ui jsdom @testing-library/react @testing-library/jest-dom @testing-library/user-event`
2. Create `vitest.config.ts`:
   ```ts
   import { defineConfig } from 'vitest/config'
   import react from '@vitejs/plugin-react'
   export default defineConfig({
     plugins: [react()],
     test: {
       environment: 'jsdom',
       setupFiles: ['./src/__tests__/setup.ts'],
       globals: true,
     },
   })
   ```
3. Create `src/__tests__/setup.ts`:
   ```ts
   import '@testing-library/jest-dom'
   ```
4. Create `src/__tests__/smoke.test.ts` — a minimal render test:
   ```ts
   import { render, screen } from '@testing-library/react'
   import App from '../App'
   test('app renders', () => {
     render(<App />)
     expect(screen.getByText('Rosetta')).toBeInTheDocument()
   })
   ```
5. Add `"test": "vitest run"` and `"test:ui": "vitest --ui"` to package.json scripts.
  </action>
  <verify>
- `npm run test` exits 0 with 1 passing test
- `vitest.config.ts` contains `jsdom` and `setupFiles`
  </verify>
  <done>`npm run test` passes; smoke test confirms React renders; jsdom environment configured.</done>
